18. Troubleshooting

Theme not showing in preview

If your preview shows a white background instead of your theme:
  • Make sure you've selected a theme (check the Themes tab in the left sidebar)
  • Try switching to a different theme and back — this forces a CSS variable refresh
  • Check that your custom theme overrides aren't set to white/empty values

YouTube video shows an error in preview

If you see "An error occurred" in the preview but it works in the editor:
  • This is a preview iframe sandbox issue — the exported site works fine
  • Export your site and open the HTML file in a browser to verify the YouTube embed works

Internal links don't work in preview

Preview simulates a static server. Internal page links should switch the preview to that page. If you get a "File not found" error, make sure the target page exists and has a valid slug.

Image not appearing

  • Check that the image URL is correct (copy it from the Asset Manager)
  • If using an external URL, make sure it's a direct image link (ends in .jpg, .png, .gif) not a page link
  • Some external sites block hotlinking — upload the image to your Asset Library instead

Exported site looks different from preview

Preview and export use the same compilation engine, so they should match. If they don't:
  • Make sure you've saved your project before exporting
  • Clear your browser cache and re-download the export
  • Open the exported style.css and verify the CSS variables are present

GitHub Pages shows 404 after publishing

GitHub Pages shows 404 after publishing

  • GitHub Pages takes 1-2 minutes to build after first publish — wait and refresh
  • Check that your repo is set to public
  • Verify Pages is enabled: Settings → Pages → Source should be "Deploy from branch: main / (root)"
  • Your URL is https://{username}.github.io/{repo-name} — check the spelling
