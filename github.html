14. 🐈‍⬛Publishing to GitHub Pages

GitHub Pages offers free hosting for static websites with optional custom domains. RetroCraft supports two levels of publishing.

Level 1: Manual Upload

No GitHub token required. After exporting your ZIP:
  1. Extract the ZIP file
  2. Create a new repository on github.com/new
  3. Upload all files (drag and drop into the repo)
  4. Go to Settings → Pages
  5. Set source to Deploy from branch, branch main, folder / (root)
  6. Wait 1-2 minutes for the build
  7. Your site is live at https://{username}.github.io/{repo-name}

Level 2: Automated Publish

RetroCraft can publish directly to GitHub using a Personal Access Token.
Setup (one-time):
  1. Go to github.com/settings/tokens
  2. Click Generate new token (classic)
  3. Give it a name like "RetroCraft"
  4. Select the repo scope (full repository access)
  5. Generate and copy the token
  6. In RetroCraft, go to Project Settings and paste the token
  7. Enter your desired repository name
  8. Click [Connect & Test] to validate

Publishing:

  1. Click [Publish to GitHub] in the top bar
  2. Enter your token password
  3. RetroCraft will: validate your token → create the repo (if needed) → push all files → enable GitHub Pages
  4. Watch the progress indicator through each step
  5. When done, your live URL is displayed with a copy button
💾!NOTE Your GitHub token is not stored in your project settings and is never logged or exposed in error messages. Treat it like a password — anyone with your token can modify your repositories.
⭐!TIP GitHub Pages can take 1-2 minutes to build after publishing. If your site shows a 404 immediately, wait a moment and refresh.
